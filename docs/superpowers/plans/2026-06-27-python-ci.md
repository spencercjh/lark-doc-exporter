# Python CI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an offline-only, Makefile-driven Python CI surface for `lark-doc-exporter` that local developers and GitHub Actions both use.

**Architecture:** Keep the verification surface intentionally small: put format/lint tool configuration in `pyproject.toml`, expose the required checks through a root `Makefile`, and make GitHub Actions do only environment bootstrap plus `make ci`. Update the README last so the documented local workflow exactly matches the implemented CI path.

**Tech Stack:** Python 3.13, `uv`, `pytest`, `ruff`, GNU `make`, GitHub Actions

---

## File Structure

- Create: `Makefile`
  Responsibility: canonical local command surface for `fmt`, `lint`, `test`, `build`, and `ci`.
- Create: `.github/workflows/python-ci.yml`
  Responsibility: GitHub bootstrap for Python 3.13 + `uv`, then `make ci`.
- Modify: `pyproject.toml`
  Responsibility: add `ruff` to the dev toolchain and record the minimal repo-local Ruff config.
- Modify: `uv.lock`
  Responsibility: lock the newly added `ruff` dependency after syncing the dev environment.
- Modify: `README.md`
  Responsibility: document the new Makefile-based local workflow and keep `doctor` clearly outside required CI.
- No new test modules
  Responsibility: this task only changes tooling, workflow, and docs surfaces, so validation is command-level (`make lint`, `make test`, `make build`, `make ci`, and workflow-file sanity checks) rather than new Python unit tests.

Expected: FAIL with `No rule to make target 'ci'`.

- [ ] **Step 2: Add Ruff to the dev toolchain and create the root Makefile**

Modify `pyproject.toml` so the dev-tooling section becomes:

```toml
[dependency-groups]
dev = [
  "pytest>=9,<10",
  "ruff>=0.12,<1",
]

[tool.ruff]
target-version = "py313"

[tool.pytest.ini_options]
pythonpath = ["src"]
```

Create `Makefile`:

```makefile
.PHONY: fmt lint test build ci

fmt:
	uv run ruff format .

lint:
	uv run ruff format --check .
	uv run ruff check .

test:
	uv run pytest -q

build:
	uv build

ci: lint test build
```

- [ ] **Step 3: Sync the environment and verify every local target works**

Run:

```bash
uv sync --python 3.13 --group dev
make lint
make test
make build
make ci
```

Expected:
- `uv sync` updates `uv.lock` and installs `ruff`
- `make lint` passes after checking format first and lint second
- `make test` passes the existing Python suite
- `make build` produces sdist/wheel artifacts
- `make ci` passes by chaining `lint`, `test`, and `build`

- [ ] **Step 4: Commit the local CI surface**

### Task 2: Add the GitHub Actions workflow that reuses `make ci`

**Files:**
- Create: `.github/workflows/python-ci.yml`

- [ ] **Step 1: Prove the workflow file is still missing**

Run:

```bash
test -f .github/workflows/python-ci.yml && echo "exists" || echo "missing"
```

Expected: `missing`

- [ ] **Step 2: Create the offline Python CI workflow**

Create `.github/workflows/python-ci.yml`:

```yaml
name: python-ci

on:
  pull_request:
  push:
    branches:
      - main
  workflow_dispatch:

jobs:
  ci:
    runs-on: ubuntu-latest

    steps:
      - name: Check out repository
        uses: actions/checkout@v4

      - name: Set up Python 3.13
        uses: actions/setup-python@v5
        with:
          python-version: "3.13"

      - name: Set up uv
        uses: astral-sh/setup-uv@v5
        with:
          enable-cache: true

      - name: Sync dev dependencies
        run: uv sync --python 3.13 --group dev

      - name: Run Makefile CI
        run: make ci
```

- [ ] **Step 3: Sanity-check the workflow file without inventing a second CI path**

Run:

```bash

grep -n "astral-sh/setup-uv@v5" .github/workflows/python-ci.yml
grep -n "uv sync --python 3.13 --group dev" .github/workflows/python-ci.yml
grep -n "make ci" .github/workflows/python-ci.yml
git diff --check
```

Expected:
- each `grep` prints the matching workflow line
- `git diff --check` prints nothing

- [ ] **Step 4: Commit the workflow**

